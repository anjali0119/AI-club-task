# Predictive Paradox: Short-Term Electricity Demand Forecasting

* A machine learning pipeline designed to predict the next hour's electricity demand (`demand_mw`) on a national grid.
* This project combines historical consumption, hourly environmental data, and annual macroeconomic indicators(urban_pop_index) from the provided datasets later merged into single dataframe; to accurately forecast demand, particularly during extreme weather events.

## Objective
Accurate electricity demand forecasting is critical for power grid stability. 
Overestimating demand leads to wasted generation, while underestimating causes severe grid instability and load shedding. 
This project works on comparing multiple regression algorithms such as linear regression, random forest regressor and LightGBM regressor to predict `t+1` demand with a Mean Absolute Percentage Error (MAPE).

## Dataset overview
The pipeline successfully merges three distinct data sources into a single dataframe'df':
1. **Core Demand Data:** Hourly grid demand (MW) and generation data from PGCB_date_power_demand.xlsx
2. **Weather Data:** Hourly environmental metrics (Apparent Temperature, Dew Point, Cloud Cover), etc from weather_data.xlsx
3. **Macroeconomic Data:** Annual indicators (Urban Population Growth) reshaped and broadcasted to hourly intervals from economic_full_1.csv

## Key Methodologies

### 1. Data Cleaning & Anomaly Handling
Before merging the data into a single dataframe for model training, each dataset underwent specific cleaning and formatting to handle missing values, correct data data types and resolve structural inconsistencies

* **Weather data cleaning:** 
Resolving the first two irrelevant rows to fix the headers; and most importantly converting **plain text(object dtype) into proper datetime & numeric format**

* **Demand data** : 
From converting the 30min intervals into hourly weighted averages to fixing missing time gaps to ensure time-lagged features(like looking exactly 24 hours back) to dropping columns with more than 70% missing data alongside removing highly correlated cols was done to reduce noise and redundancy.

* **Economic data:** 
Reshaping data from **wide format to a standard layout** using .melt() and .pivot() was crucial step in this dataset before eda; also dropping other cols and keeping necessary- urban population growth index was done alongside dropping other cols wiht high missing values.

* **Final Merge** : Combining these three datasets using **year*** to form a single dataframe for model training

### 2. Outlier handling & visulaization
* **Removing uncorrelated variables** :
Dropped features like 'sunshine' and 'precipitation' that showed near-zero linear correlation with power demand to reduce complexity

* **visualising anomalies** :
Used a **side-by-side boxplot and scatterplot** to identify extreme, undocumented spikes in the dataset.

* **data capping(clipping)** :
1. Initial exploratory data analysis (EDA) revealed that the raw grid generation data contained extreme, non-physical hardware anomalies( sensor multiplier glitches recording over 121,000 MW).

2. Because **electricity demand is a highly skewed, non-Gaussian distribution, traditional parametric anomaly detection (like the 3-Sigma Z-Score) fails entirely—the magnitude of the 121,000 MW anomalies artificially inflates the standard deviation, rendering Z-score thresholds useless**.

3. To prevent these extreme outliers from skewing the model, the demand_mw values were **capped at the 1st and 99th percentiles using np.clip().** This created a stable demand_clean column.

### 3. Feature Engineering
Shifted the clean demand data up by one row (shift(-1)) to create the target variable; 

* **Historical Inertia:** 
created exact lag features (1 hour, 24 hours, and 1 week prior) and applied 6-hour and 24-hour rolling averages so the model can learn daily and weekly usage cycles.

* **time-based features** :
Extracted hour, month, and day of the week. Added binary indicators for weekends and evening peak hours (5 PM – 11 PM) when demand typically surges.

* **Cyclical Indicators:** 
Transformed wind direction from standard degrees into sine and cosine components. This ensures the model understands wind direction as a continuous circle rather than a linear number.

* **Interaction Features:** 
Created high-value compound features like "pop_heat_stress" (Apparent Temperature Squared × Urban Population Growth),"muggy index" (apparent temp × dew point); to map macroeconomic trends to short-term cooling demand. 

* **final cleanup** :
Dropped all rows containing missing values (NaN) generated by the lagging and rolling window operations to ensure a clean dataset for training

### 3. Model training & Data splitting
* **chronological data split** :
To prevent data leakage and accurately simulate real-world forecasting, the data was split chronologically rather than randomly. Data from 2023 and earlier was used for the training set, while data from 2024 onwards was reserved strictly for the testing set.

* **Algorithm selection:** 
three different models were trained to compare performance:
1] Linear Regression
2] Random Forest Regressor
3] LightGBM

### 4. Model evaluation & results:
The models were evaluated using Mean Squared Percentage Error (MSPE) as the primary evaluation metric, alongside MAPE, MAE, and RMSE. The results showed a clear progression in accuracy, with LightGBM outperforming the baselines:

**Linear Regression Baseline: * MSPE: 0.757%,  RMSE: 576.91 MW**

**Random Forest Regressor: * MSPE: 0.641%, RMSE: 485.69 MW**

**LightGBM (Winning Model): * MSPE: 0.581%, RMSE: 413.90 MW, MAE: 204.76 MW**

### 5. Results & Performance
Evaluated on the hold-out 2024 test set, the model achieved highly stable, production-ready metrics by "LightGBM model:

* **MAPE:** 2.014 %
* **MSPE:** 0.580 %
* **MAE:** ~203 MW
* **RMSE:** ~421 MW

Also during LightGBM training, the logs indicated "No further splits with positive gain." This simply confirms the model efficiently reached optimal tree depths early, preventing unnecessary complexity and overfitting while achieving the best MSPE.

### 6. Feature importance analysis
* To understand how the final LightGBM model makes its predictions, we analyzed its feature importance based on the **how often the feature was used to build decision trees**

* the top drivers were -
1] **Time(hour)** :
The specific time of day (hour) was the most heavily utilized feature, proving that electricity consumption is fundamentally dictated by daily human routines like peak usuage in mornings, evening peaks, etc

2] **Recent history & cyclical trends** :
* The immediate previous hour (lag_current_hour) was the second most critical feature, confirming strong short-term continuity in demand; 
* The model also relied heavily on exact historical cycles, frequently splitting on data from exactly one day ago (lag_24_hours_ago) and one week ago (lag_1_week_ago), alongside rolling averages (rolling_mean_24h, rolling_mean_6h)

3] **weather & interactions** :
* The model fine-tuned its predictions using weather data, apparent_temp and dew_point as these were the most important environmental factors. 
* whereas the custom engineered features like pop_heat_stress and muggy_idx ranked high, validating that capturing the non-linear relationship between population growth and extreme weather improves forecasting accuracy.

* Overall, the LightGBM model successfully learned that power demand is a highly seasonal and cyclical time-series problem, with weather acting as a secondary multiplier

### 7. Conclusion 
* the LightGBM model successfully forecasted next-hour electricity demand, achieving an exceptionally low Mean Squared Percentage Error (MSPE) of 0.581%.

* By accurately predicting demand based on time cycles and weather-driven heat stress, grid operators can proactively optimize power generation.
